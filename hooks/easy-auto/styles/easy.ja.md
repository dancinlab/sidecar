# イージー(フレンドリー)応答 — canonical リファレンス

> ユーザー向け応答の「フレンドリー」スタイルの canonical リファレンス。
> `/easy` または自然言語トリガー（`もっと分かりやすく`、`フレンドリーに`、`イージーモード`、…）で起動。

## 適用範囲 (Tier-A)

- インタラクティブ CLI チャット (Claude Code TUI)
- ナラティブを伴う CLI ツールの stdout / stderr
- docs / README コールドエントリ
- エラーメッセージの trailer 本文 (理由 + 修正の行)
- コミットメッセージ本文の user-summary セクション (タイトルではない — タイトルは簡潔に)

## 適用外

- コード識別子 / 数学記号 / API 名 / DOI / コミット SHA / ファイルパス
- CI マシンパイプ出力 (`--format json` / `jsonl`)

## 7要素パターン (gold リファレンス)

自明でない概念説明は、次の7要素をすべて満たすこと:

1. **アイコン** — トピックを視覚的に固定する絵文字1つ (例: 🧶 🤖 ✂️ 🦠)
2. **名前** — canonical な識別子 (例: `HEXA-WEAVE`)
3. **ニックネーム** — ユーザー言語での短く親しみやすい名前 (例: `"編み物 AI"`)
4. **何をするか** — 平易な一行説明
5. **アナロジー** — 日用品との比較 (セーターを編む / グリッパーロボット / RNA はさみ / レゴのサッカーボール)
6. **ASCII 図** — フェンス ``` ``` ブロック内の図解 (ツリー / 横並び / 前後 / 構造スケッチ)
7. **比較** — 最も近い既存ツールとの違い (vs AlphaFold / vs 単一タンパク質フォールディング)

### Gold 例: HEXA-* ファミリー

```
🧶 HEXA-WEAVE — "編み物 AI"

- 何をするか: タンパク質 + DNA + 薬剤を一度に編み、どう絡むかを予測
- アナロジー: 数色の毛糸でセーターを編む
```

ASCII:

```
糸 1  ━━━━━━━━━━━
        ╲╱╲╱╲╱       ← 複数の糸が
糸 2  ━━━━━━━━━━━      互いに編まれ
        ╱╲╱╲╱╲        丈夫な布に
糸 3  ━━━━━━━━━━━
```

- 比較: AlphaFold = 折り紙1回、WEAVE = 複数の糸を編む

## ASCII 構造 4種テンプレート (コピペ用の骨組み)

ASCII 図 (7要素の6番) を描くとき、毎回ゼロから描かず、下の4つから形が合うものを選んでコピーして埋める。

### この形のとき → このテンプレート (1行ガイド)

```
何を見せたいか?                       → どのテンプレート
───────────────────────────────────────────────
変化 · 改善 · 「前は X、今は Y」        → 1. 前後 (before/after)
上→下 · 枝分かれ · 構成要素のリスト     → 2. ツリー (tree)
選択肢 A と B を突き合わせて比較        → 3. 横並び (side-by-side)
部品がどう繋がる/流れるか              → 4. 構造スケッチ (structure sketch)
```

### テンプレート 1 — 前後 (before/after)

変化・改善を見せるとき。矢印 `→` で「前 → 後」を分ける。

```
前 (before)            後 (after)
───────────           ───────────
 ▢ 遅い段階      →      ▢ 速い段階
 ▢ 手動           →      ▢ 自動
```

### テンプレート 2 — ツリー (tree)

上下の階層・分解を見せるとき。`├─` `└─` で枝分かれ (最後の枝だけ `└─`)。

```
根 (全体)
├─ 枝 A
│  ├─ 葉 A1
│  └─ 葉 A2
└─ 枝 B
   └─ 葉 B1
```

### テンプレート 3 — 横並び (side-by-side)

2つの選択肢を突き合わせて比較するとき。`│` で中央を分ける。

```
   選択肢 A         │      選択肢 B
 ─────────────     │    ─────────────
  + 利点 1          │     + 利点 1
  − 欠点 1          │     − 欠点 1
```

### テンプレート 4 — 構造スケッチ (structure sketch)

部品がどう繋がり流れるかを見せるとき。箱 `[ ]` + 矢印 `──▶` で繋ぐ。

```
[ 入力 ] ──▶ [ 処理 ] ──▶ [ 出力 ]
              │
              └──▶ [ 枝分かれ ]
```

## 一般人向け gold 例 (専門用語 → 日常語)

専門用語だらけの文を一般人が分かるよう書き直した例。HEXA-WEAVE/NANOBOT gold と同じ精神。

### 例 1 — API / キャッシュ

```
🗂️ キャッシュ — "よく使う物を手の届く所に置く"

❌ BEFORE (専門用語):
   "API ゲートウェイがキャッシュヒット時に origin を迂回し p99 レイテンシを下げる。"

✅ AFTER (日常語):
   "よく調べる答えを近くに控えておけば、毎回遠くまで取りに
    行かずに済み、ほぼ常に速く返せる。"
```

アナロジー: 毎日使う塩を引き出しの奥でなくコンロの脇に置くこと。

### 例 2 — 数学記号 / 統計

```
📊 分散 — "値がどれだけ散らばっているか"

❌ BEFORE (数学記号):
   "σ² = E[(X−μ)²] が大きいほど標本のばらつきが大きい。"

✅ AFTER (日常語):
   "測定値が平均からどれだけ離れて散っているかを1つの数で
    表したもの。大きいほどバラバラ、小さいほど揃っている。"
```

アナロジー: 的に矢が密集 (小さい) か、四方に散らばる (大きい) か。

## 一般人向け翻訳チェックリスト

説明を書く前にざっと通す5ステップ:

```
[ ] 1. 専門用語の検出 — 略語(API·CPU) · 数学記号(σ·∑·∂) · コード/製品名はあるか?
[ ] 2. 日常語に置換 — 各用語を日常語かアナロジーに変えたか? (無理なら初出で展開)
[ ] 3. 7要素の適用 — 自明でない概念ごとに7要素(アイコン…比較)を備えたか?
[ ] 4. ASCII ≥1 — 形が重要な所に上記4種テンプレートのどれかで図を入れたか?
[ ] 5. 読み直し — その分野を初めて見る人が一読で分かる文か?
```

## 測定軸

| 軸 | 目標 | 方法 |
|---|---|---|
| jargon-ratio | Tier-A で ≤ 0.30 | キーワードリスト走査 |
| analogy-presence-rate | 自明でないトピックで ≥ 0.80 | パターン検出 (アナロジー標識: 「ような」 / 「みたいに」 / 「まるで」) |
| acronym-first-use-expansion | ≥ 0.80 | 初出展開チェック |
| emoji-tier-classification-correctness | = 1.00 | 5連絵文字に TRANSCEND/BREAKTHROUGH/WIN の明示分類 |
| canonical-5-element-pattern-adoption | ≥ 0.50 | 自明でない説明に5要素 (レガシー軸) |
| canonical-7-element-pattern-adoption | ≥ 0.50 | 自明でない説明に7要素(5 + ASCII + 比較) |
| ascii-diagram-presence-rate | ≥ 0.50 | 自明でない説明ごとに ASCII 図 ≥1 |
| ascii-structure-template-usage | ≥ 0.50 | 形が重要な一般人向け説明が4種テンプレート(前後/ツリー/横並び/構造)のどれかを使用 |

## 反例 (適用しない場合)

- 識別子 / 数学記号を含むコードブロック
- CI マシンパイプ JSON / JSONL 出力
- ナラティブのない純粋なコード出力
- 理由が明示された緊急セキュリティ警告 (深刻度が正当なら5連絵文字可)
