# 이지(친근) 응답 — canonical 레퍼런스

> 사용자 대면 응답의 "친근한" 스타일 canonical 레퍼런스.
> `/easy` 또는 자연어 트리거(`친근하게`, `쉽게 설명해줘`, `이지 모드`, …)로 활성화.

## 적용 범위 (Tier-A)

- 인터랙티브 CLI 채팅 (Claude Code TUI)
- 내러티브가 있는 CLI 도구 stdout / stderr
- docs / README 콜드 엔트리
- 에러 메시지 trailer 본문 (원인 + 해결 줄)
- 커밋 메시지 본문의 user-summary 섹션 (제목 아님 — 제목은 간결 유지)

## 적용 제외

- 코드 식별자 / 수학 기호 / API 이름 / DOI / 커밋 SHA / 파일 경로
- CI 머신 파이프 출력 (`--format json` / `jsonl`)

## 7-요소 패턴 (gold 레퍼런스)

사소하지 않은 개념 설명은 다음 7요소를 모두 갖춰야 한다:

1. **아이콘** — 주제를 시각적으로 고정하는 이모지 1개 (예: 🧶 🤖 ✂️ 🦠)
2. **이름** — canonical 식별자 (예: `HEXA-WEAVE`)
3. **별칭** — 사용자 언어로 된 짧고 친근한 이름 (예: `"뜨개질 AI"`)
4. **하는 일** — 한 줄 평이한 설명
5. **비유** — 일상 사물 비교 (스웨터 짜기 / 집게 로봇 / RNA 가위 / 레고 축구공)
6. **ASCII 다이어그램** — 펜스 ``` ``` 블록의 시각 도식 (트리 / 나란히 / 전후 / 구조 스케치)
7. **비교** — 가장 가까운 기존 도구와 어떻게 다른지 (vs AlphaFold / vs 단일 단백질 폴딩)

### Gold 예시: HEXA-* 패밀리

```
🧶 HEXA-WEAVE — "뜨개질 AI"

- 하는 일: 단백질 + DNA + 약물을 한 번에 짜서 어떻게 얽히는지 예측
- 비유: 여러 색실로 스웨터 짜기
```

ASCII:

```
실 1  ━━━━━━━━━━━
        ╲╱╲╱╲╱       ← 여러 가닥이
실 2  ━━━━━━━━━━━      서로 짜여
        ╱╲╱╲╱╲        탄탄한 천
실 3  ━━━━━━━━━━━
```

- 비교: AlphaFold = 종이접기 1개, WEAVE = 여러 가닥 짜기

---

```
🤖 HEXA-NANOBOT — "분자 로봇팔"

- 하는 일: 분자가 움직이는 방식 설계 (열고 닫고, 잡고 놓고)
- 비유: 매우 작은 집게 로봇
```

ASCII:

```
   ╱ ╲              ╱╲
  │   │     →      │ │   ← 분자 잡음
   ╲ ╱              ╲╱
   (열림)          (닫힘)
```

- 핵심: DNA-origami 같은 걸로 "스위치" 만들기

### Gold 비교 예시: FOLD vs WEAVE

| 축 | FOLD (종이접기) | WEAVE (뜨개질) |
|---|---|---|
| 행위 | "접기" | "짜기" |
| 재료 | 끈 1개 | 실 여러 가닥 |
| 결과물 | 종이학 | 스웨터 · 바구니 |
| 비교 도구 | AlphaFold (2020~) | HEXA-WEAVE (2026~) |

## ASCII 구조 4종 템플릿 (복붙용 뼈대)

ASCII 다이어그램(7요소 중 6번)을 그릴 때, 매번 처음부터 그리지 말고 아래 4개 중 모양이 맞는 걸 골라 복사해서 채운다.

### 이 모양일 때 → 이 템플릿 (1줄 가이드)

```
무엇을 보여주고 싶은가?            → 어떤 템플릿
─────────────────────────────────────────────
바뀜 · 개선 · "전에는 X, 이제는 Y"  → 1. 전후 대조 (before/after)
상위→하위 · 갈라짐 · 구성요소 목록   → 2. 트리 (tree)
선택지 A 와 B 를 맞대고 비교        → 3. 나란히 (side-by-side)
부품들이 어떻게 연결/흐르는가        → 4. 구조 스케치 (structure sketch)
```

### 템플릿 1 — 전후 대조 (before/after)

변화·개선을 보여줄 때. 화살표 `→` 로 "전 → 후" 를 가른다.

```
전 (before)            후 (after)
───────────           ───────────
 ▢ 느린 단계     →      ▢ 빠른 단계
 ▢ 수동           →      ▢ 자동
```

### 템플릿 2 — 트리 (tree)

상하 위계·분해를 보여줄 때. `├─` `└─` 로 가지를 친다 (마지막 가지만 `└─`).

```
뿌리 (전체)
├─ 가지 A
│  ├─ 잎 A1
│  └─ 잎 A2
└─ 가지 B
   └─ 잎 B1
```

### 템플릿 3 — 나란히 (side-by-side)

두 선택지를 맞대어 비교할 때. `│` 로 가운데를 가른다.

```
   선택지 A         │      선택지 B
 ─────────────     │    ─────────────
  + 장점 1          │     + 장점 1
  − 단점 1          │     − 단점 1
```

### 템플릿 4 — 구조 스케치 (structure sketch)

부품이 어떻게 이어지고 흐르는지 보여줄 때. 상자 `[ ]` + 화살표 `──▶` 로 연결한다.

```
[ 입력 ] ──▶ [ 처리 ] ──▶ [ 출력 ]
              │
              └──▶ [ 곁가지 ]
```

## 일반인 gold 예시 (전문용어 → 생활어)

전문용어가 잔뜩 든 문장을 일반인이 알아듣게 다시 쓴 예시. HEXA-WEAVE/NANOBOT gold 와 같은 정신.

### 예시 1 — API / 캐시

```
🗂️ 캐시 — "자주 쓰는 물건 손 닿는 곳에 두기"

❌ BEFORE (전문용어):
   "API 게이트웨이가 캐시 히트 시 origin 을 우회해 p99 레이턴시를 낮춘다."

✅ AFTER (생활어):
   "자주 찾는 답을 미리 가까이 적어두면, 매번 멀리 가서 찾지
    않아도 돼서 거의 항상 빨리 응답한다."
```

비유: 부엌에서 매일 쓰는 소금을 서랍 깊이가 아니라 가스레인지 옆에 두는 것.

### 예시 2 — 수학 기호 / 통계

```
📊 분산 — "값들이 얼마나 흩어졌나"

❌ BEFORE (수학 기호):
   "σ² = E[(X−μ)²] 가 클수록 표본 분포의 산포가 크다."

✅ AFTER (생활어):
   "측정값들이 평균에서 얼마나 멀리 퍼져 있는지를 한 숫자로
    나타낸 것. 클수록 들쭉날쭉, 작을수록 고르다."
```

비유: 과녁에 화살이 빽빽이 모였나(작음), 사방에 흩어졌나(큼).

## 일반인-번역 체크리스트

설명을 쓰기 전에 빠르게 훑는 5단계:

```
[ ] 1. 전문용어 탐지 — 약어(API·CPU) · 수학 기호(σ·∑·∂) · 코드/제품 이름이 있나?
[ ] 2. 생활어로 치환 — 각 용어를 일상 단어 또는 비유로 바꿨나? (못 바꾸면 첫 등장에 풀어쓰기)
[ ] 3. 7요소 적용 — 사소하지 않은 개념마다 7요소(아이콘…비교)를 갖췄나?
[ ] 4. ASCII ≥1 — 모양이 중요한 곳에 위 4종 템플릿 중 하나로 다이어그램을 넣었나?
[ ] 5. 다시 읽기 — 그 분야를 처음 보는 사람이 한 번에 이해할 문장인가?
```

## 측정 축

| 축 | 목표 | 방법 |
|---|---|---|
| jargon-ratio | Tier-A에서 ≤ 0.30 | 키워드 목록 스캔 |
| analogy-presence-rate | 사소하지 않은 주제에서 ≥ 0.80 | 패턴 탐지 (비유 마커: "같은" / "처럼" / "마치") |
| acronym-first-use-expansion | ≥ 0.80 | 첫 등장 풀어쓰기 확인 |
| emoji-tier-classification-correctness | = 1.00 | 5개 이모지에 TRANSCEND/BREAKTHROUGH/WIN 명시 분류 |
| canonical-5-element-pattern-adoption | ≥ 0.50 | 사소하지 않은 설명에 5요소 존재 (레거시 축) |
| canonical-7-element-pattern-adoption | ≥ 0.50 | 사소하지 않은 설명에 7요소(5 + ASCII + 비교) 존재 |
| ascii-diagram-presence-rate | ≥ 0.50 | 사소하지 않은 설명당 ASCII 다이어그램 ≥1 |
| ascii-structure-template-usage | ≥ 0.50 | 모양이 중요한 일반인 설명이 4종 템플릿(전후/트리/나란히/구조) 중 하나를 사용 |

## 반례 (적용하지 않을 때)

- 식별자 / 수학 기호가 든 코드 블록
- CI 머신 파이프 JSON / JSONL 출력
- 내러티브 없는 순수 코드 출력
- 사유가 명시된 긴급 보안 경보 (심각도 정당화 시 5개 이모지 허용)
