description: Show or set the wilson-pool host roster. Heavy Bash commands route via ssh to a roster host — macOS-only / Linux-only commands go to a host of that platform, otherwise round-robined. workdir may be `auto` (mirror the current project path) or an explicit path, with an optional per-host override and a `fallback`. `autosync on` rsyncs the local project to the target host before each routed command (so a continuously-changing tree stays fresh and the workdir is created if absent); `autosync mirror` adds rsync --delete; `autosync off` just pre-flight-checks the workdir exists and skips routing to hosts that lack it. A host tagged `sudo` (on `add`, or via `sudo <host> on`) auto-prefixes root-needing commands (apt, dpkg, yum, dnf, pacman, systemctl, ldconfig, setcap) with `sudo` when they route there — and those commands route on their own even outside the heavy `patterns`. External (off-LAN) access via tailscale: `tailnet` lists tailnet nodes (name/OS/online/in-pool); `add <node> [tailscale|ssh]` registers with an optional transport override and auto-detects the platform from the tailnet; `status` adds a live per-host reachability probe + transport-in-use on top of `show`. Default transport is `auto` (tailscale ssh when a local tailscale daemon is up, else plain ssh).
