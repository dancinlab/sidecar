## 평이-초록

💸 **비용 라우팅(cost routing)** — 별칭 *"똑똑한 택배 분류기"*

- 하는 일: 들어온 질문 하나하나를 보고, **싼 일꾼(작은 모델)** 으로 충분한지
  **비싼 일꾼(큰 모델)** 이 필요한지 자동으로 골라 보낸다.
- 비유: 우체국에서 가벼운 엽서는 일반 우편으로, 깨지기 쉬운 큰 짐만 특급으로
  보내는 분류 직원.

원래 초록은 이렇게 시작합니다(전문용어 버전):

> "쿼리별 난이도 추정에 따라 inference 를 tiered backend 로 라우팅하면, 품질
>  저하 없이 평균 토큰당 비용을 절감한다."

같은 말을 생활어로:

> "질문이 쉬운지 어려운지 먼저 가늠해서, 쉬운 건 값싼 모델에, 어려운 것만
>  비싼 모델에 보낸다. 그러면 답의 품질은 그대로인데 전체 비용은 줄어든다."

여기서 *모델(model)* = AI 일꾼, *토큰(token)* = 글자 묶음(요금 단위, 통화료의
'초'처럼 쓴 만큼 돈이 나가는 단위)이라고 보면 됩니다.

## ASCII 그림 스케치

**그림 1 — 분류기가 하는 일 (구조 스케치, 템플릿 ④)**

```
[ 질문 ] ──▶ [ 난이도 가늠 ] ──┬──▶ [ 싼 모델 ]  ──▶ [ 답 ]
                               │
                               └──▶ [ 비싼 모델 ] ──▶ [ 답 ]
                                     (어려울 때만)
```

**그림 2 — 비용, 전과 후 (전후 대조, 템플릿 ①)**

```
전 (전부 비싼 모델)        후 (똑똑한 분류)
──────────────────        ──────────────────
 ▢ 100원/질문         →    ▢ 평균 62원/질문
 ▢ 품질 좋음          →    ▢ 품질 그대로
                          (= 38% 절감)
```

**그림 3 — 두 방식 맞대보기 (나란히, 템플릿 ③)**

```
   전부 비싼 모델        │      똑똑한 분류기
 ─────────────────     │    ─────────────────
  + 항상 최고 품질       │     + 품질 동일
  − 항상 비쌈           │     + 비용 38%↓
                       │     − 가늠이 가끔 틀림
```

## 비유로 보는 방법 & 발견

**방법 (HOW) — "우체국 분류대"**
질문이 들어오면 분류 직원이 먼저 무게를 잰다(난이도 가늠). 가벼운 엽서는
일반 우편(싼 모델), 깨지기 쉬운 큰 짐만 특급(비싼 모델). 핵심은 **무게 재는
일 자체는 아주 싸다** 는 것 — 그래서 분류에 드는 비용보다 아낀 비용이 훨씬
크다. 논문은 이 "재기 → 보내기"를 실제로 돌려보고 질문당 평균 비용을 셌다.

**발견 (WHAT) — "같은 품질, 더 싼 영수증"**
영수증을 더해보니(비용 합산 항등식: 전체 = Σ 질문별 비용), 똑똑한 분류기는
전부 비싼 모델을 쓸 때보다 **평균 38% 싸면서 답의 품질은 같았다**. 즉
"품질을 깎아서 싸진 것"이 아니라 "안 써도 될 비싼 일꾼을 안 불러서" 싸진
것이다 — 이게 이 논문의 핵심 측정값이다.

## 이 논문이 한 일

이 논문은 "AI에게 질문할 때, 쉬운 질문까지 비싼 모델한테 보내느라 돈을
낭비하고 있지 않나?"를 물었다. 그래서 질문마다 난이도를 싸게 가늠해 쉬운 건
값싼 모델, 어려운 것만 비싼 모델로 보내는 '분류기'를 만들어 실제로 돌리고,
영수증을 한 건 한 건 더해(비용 합산 항등식) 평균 비용을 쟀다. 결과는 답의
품질을 떨어뜨리지 않고도 평균 38%를 아꼈다는 것 — 즉 비싼 모델은 정말 필요할
때만 부르면 된다는, 돈으로 검증된 발견이다.
