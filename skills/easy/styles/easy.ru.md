# Easy (дружелюбный) ответ — канонический эталон

> Канонический эталон «дружелюбного» стиля ответов пользователю.
> Активируется через `/easy` или естественно-языковой триггер (`попроще`, `по-дружелюбнее`, `Easy mode`, …).

## В области применения (Tier-A)

- Интерактивный CLI-чат (Claude Code TUI)
- stdout / stderr CLI-инструментов с повествованием
- docs / README, «холодный» вход
- тело-trailer сообщения об ошибке (причина + строки исправления)
- секция user-summary в теле коммит-сообщения (НЕ заголовок — заголовок краткий)

## Вне области применения

- Идентификаторы кода / матсимволы / имена API / DOI / SHA коммита / пути файлов
- Машинный вывод CI (`--format json` / `jsonl`)

## Паттерн из 7 элементов (золотой эталон)

Каждое нетривиальное объяснение концепции должно содержать эти 7 элементов:

1. **Иконка** — один эмодзи, визуально закрепляющий тему (напр. 🧶 🤖 ✂️ 🦠)
2. **Имя** — канонический идентификатор (напр. `HEXA-WEAVE`)
3. **Прозвище** — короткое дружелюбное имя на языке пользователя (напр. `"вязальный ИИ"`)
4. **Что делает** — одна простая строка
5. **Аналогия** — сравнение с бытовым предметом (вязание свитера / робот-захват / РНК-ножницы / лего-мяч)
6. **ASCII-схема** — схема в огороженном ``` ``` блоке (дерево / рядом / до-после / структурный набросок)
7. **Сравнение** — чем отличается от ближайшего существующего инструмента (vs AlphaFold / vs фолдинг одного белка)

### Золотой пример: семейство HEXA-*

```
🧶 HEXA-WEAVE — "вязальный ИИ"

- что делает: предсказывает, как белки + ДНК + лекарства сплетаются вместе
- аналогия: вязать свитер из нескольких цветных нитей
```

ASCII:

```
нить 1 ━━━━━━━━━━━
         ╲╱╲╱╲╱       ← несколько нитей
нить 2 ━━━━━━━━━━━       переплетаются в
         ╱╲╱╲╱╲         прочную ткань
нить 3 ━━━━━━━━━━━
```

- сравнение: AlphaFold = одно оригами; WEAVE = плетение многих нитей

---

```
🤖 HEXA-NANOBOT — "молекулярная рука-робот"

- что делает: проектирует движение молекулы (открыть/закрыть, взять/отпустить)
- аналогия: очень маленький робот-захват
```

ASCII:

```
   ╱ ╲              ╱╲
  │   │     →      │ │   ← захватывает молекулу
   ╲ ╱              ╲╱
 (открыт)        (закрыт)
```

- суть: сделать «переключатель» из чего-то вроде DNA-origami

### Золотой пример сравнения: FOLD vs WEAVE

| Ось | FOLD (оригами) | WEAVE (вязание) |
|---|---|---|
| Действие | «складывать» | «плести» |
| Материал | одна нить | много нитей |
| Результат | бумажный журавлик | свитер · корзина |
| Сравнимый инструмент | AlphaFold (2020~) | HEXA-WEAVE (2026~) |

## 4 ASCII-шаблона структуры (скелеты для копирования)

Когда рисуете ASCII-схему (элемент 6 из 7), не начинайте с нуля — выберите из этих 4 тот, чья форма подходит, скопируйте и заполните.

### Когда выглядит так → берите этот шаблон (однострочный гид)

```
Что хотите показать?                       → какой шаблон
───────────────────────────────────────────────────────
изменение · улучшение · «было X, стало Y»   → 1. до/после (before/after)
сверху→вниз · ветвление · список частей      → 2. дерево (tree)
поставить вариант A и B друг против друга     → 3. рядом (side-by-side)
как части соединяются / текут                → 4. набросок структуры (structure sketch)
```

### Шаблон 1 — до/после (before/after)

Чтобы показать изменение или улучшение. Стрелка `→` делит «до → после».

```
до (before)            после (after)
───────────           ───────────
 ▢ медленный шаг  →     ▢ быстрый шаг
 ▢ вручную        →     ▢ автоматически
```

### Шаблон 2 — дерево (tree)

Чтобы показать иерархию или декомпозицию. Ветвите через `├─` и `└─` (только последняя ветка — `└─`).

```
корень (целое)
├─ ветка A
│  ├─ лист A1
│  └─ лист A2
└─ ветка B
   └─ лист B1
```

### Шаблон 3 — рядом (side-by-side)

Чтобы сравнить два варианта лицом к лицу. `│` делит середину.

```
   вариант A         │      вариант B
 ─────────────       │    ─────────────
  + плюс 1            │     + плюс 1
  − минус 1           │     − минус 1
```

### Шаблон 4 — набросок структуры (structure sketch)

Чтобы показать, как части соединяются и текут. Соединяйте блоки `[ ]` стрелками `──▶`.

```
[ вход ] ──▶ [ обработка ] ──▶ [ выход ]
              │
              └──▶ [ ответвление ]
```

## Золотые примеры для непрофессионала (жаргон → обычные слова)

Перегруженное жаргоном предложение, переписанное так, чтобы понял непрофессионал — в том же духе, что золотые примеры HEXA-WEAVE/NANOBOT.

### Пример 1 — API / кэш

```
🗂️ Кэш — "держать то, чем часто пользуешься, под рукой"

❌ BEFORE (жаргон):
   "API-шлюз обходит origin при попадании в кэш, снижая p99-latency."

✅ AFTER (обычные слова):
   "Мы заранее записываем ответы, которые ищем чаще всего, и держим
    их рядом, чтобы не бегать за ними далеко каждый раз — почти всегда
    быстрый ответ."
```

Аналогия: соль, которой пользуешься каждый день, ставишь у плиты, а не в глубину ящика.

### Пример 2 — матсимволы / статистика

```
📊 Дисперсия — "насколько разбросаны числа"

❌ BEFORE (матсимволы):
   "Чем больше σ² = E[(X−μ)²], тем больше разброс выборки."

✅ AFTER (обычные слова):
   "Одно число о том, как далеко измерения сидят от среднего.
    Больше = вразброс; меньше = плотно сгруппированы."
```

Аналогия: стрелы кучно в яблочко (мало) против разбросанных повсюду (много).

## Чек-лист перевода для непрофессионала

Быстрый проход из 5 шагов перед написанием объяснения:

```
[ ] 1. Найти жаргон — есть аббревиатуры(API·CPU) · матсимволы(σ·∑·∂) · имена кода/продукта?
[ ] 2. Заменить обычными словами — каждый термин заменён простым словом или аналогией? (если нельзя — раскрыть при первом появлении)
[ ] 3. Применить 7 элементов — каждое нетривиальное понятие несёт все 7 (иконка…сравнение)?
[ ] 4. ASCII ≥1 — где важна форма, добавлена схема одним из 4 шаблонов выше?
[ ] 5. Перечитать — поймёт ли с одного прохода тот, кто видит эту область впервые?
```

## Оси измерения

| Ось | Цель | Метод |
|---|---|---|
| jargon-ratio | ≤ 0.30 на Tier-A | Скан по списку ключевых слов |
| analogy-presence-rate | ≥ 0.80 на нетривиальных темах | Детекция паттерна (маркеры аналогии: «как» / «словно» / «вроде») |
| acronym-first-use-expansion | ≥ 0.80 | Проверка раскрытия при первом появлении |
| emoji-tier-classification-correctness | = 1.00 | Явная классификация TRANSCEND/BREAKTHROUGH/WIN на 5-кратном |
| canonical-5-element-pattern-adoption | ≥ 0.50 | Наличие 5 элементов в нетривиальных объяснениях (легаси-ось) |
| canonical-7-element-pattern-adoption | ≥ 0.50 | Наличие 7 элементов (5 + ASCII + сравнение) в нетривиальных объяснениях |
| ascii-diagram-presence-rate | ≥ 0.50 | ≥1 ASCII-схема на нетривиальное объяснение |
| ascii-structure-template-usage | ≥ 0.50 | Объяснения для непрофессионала, где важна форма, используют один из 4 шаблонов (до-после/дерево/рядом/структура) |

## Контрпример (когда НЕ применять)

- Блоки кода с идентификаторами / матсимволами

- Чистый вывод кода без повествования
- Срочное оповещение безопасности с заявленным обоснованием (при оправданной критичности 5-кратный эмодзи допустим)
