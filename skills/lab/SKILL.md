## 4축 auto-decide (mid-run 결정)

A campaign run will hit forks that need a *choice between options* mid-flight (live lesson: an RTSC run surfaced a strategy fork — clean-reset vs targeted-fix vs let-it-settle — to the user mid-run). When the fork is **technically-resolvable** (every option is viable; the call is a trade-off, not a missing human input), `/lab` AUTO-PICKS on the canonical 4 axes and continues, instead of stalling. This subsection is the shared rule referenced by the `next` / `auto` / `drive` / `pursue` verbs above; every multi-option fork in those verbs routes through it.

**Axes (recommend.tape r1 — cited verbatim, not redefined):**
- ① **완성도 (complete)** — the most robust option: handles edge-cases · fewest unhandled failure modes · highest coverage.
- ② **단순 (simple)** — the Occam option: fewest moving parts · least new surface · easiest to hold in one head.
- ③ **안전 (safe)** — the minimal-blast-radius option: smallest irreversible footprint · easiest to roll back · least collateral.
- ④ **표준 (std)** — the convention-fit option: matches the established pattern in this repo/domain · least surprise to a future reader.

**Rule (@L2):** for a fork with >1 technically-viable option, score each option **1–5** on 완성도·단순·안전·표준, take the **weighted average** (default weights `1:1:1:1`; inherit session weights if the user set any via `recommend` / `sbs`, e.g. `safe=2`), and **pick the max**. **Tie → the 안전(safe) champion wins** — mid-campaign, prefer to minimize blast radius / pick the most reversible option. This is the auto-mode collapse of recommend.tape's 4-champion box (r2/r3): instead of surfacing four champions for the user to choose, score-and-collapse to one pick.

**Surface (@L4):** render the pick as **ONE collapsed line** (recommend.tape r2/r3 auto-mode style — NOT the 4-line box):

```
🤖 4축 auto-pick: <option> (완성도=X 단순=Y 안전=Z 표준=W · weighted=<sum>)
```

so the user sees the trade-off post-hoc and can override on the next turn.

**Human-gate carve-out (@L3) — the auto-decide NEVER overrides the autonomy invariant.** These are the ONLY things that still HALT the loop for a human; everything else auto-picks:
1. **`gated:<human-only-input>`** — a genuine human-only input, in exactly three classes:
   - **credential** — a secret/key/token the human must supply;
   - **irreversible/destructive** — an action with no rollback (data deletion, a spend commitment, an external publish);
   - **genuine open-ended design decision** — an unbounded "what should we even build?" judgment with no enumerable option set.
2. **budget cap (g64)** — a declared spend cap breach still halts regardless.

A fork with an **enumerable set of viable options** is NOT a human gate — it is technically-resolvable and auto-picks via the rule above. A science-direction judgment with parallelizable branches is also not a gate (pursue runs all viable branches in parallel; a single bounded mutually-exclusive fork auto-picks here).

## upstream 자동 dispatch (g59 무조건 강화)

The g59 upstream-reflex used to STOP at filing a `sidecar handoff` note — the gap stayed unfixed until a human picked it up. **1.5.0 strengthens g59 so a detected upstream-fixable gap AUTO-SHIPS the fix, not just a note.** When a campaign run hits a gap the tooling should have handled itself — a `hexa cloud`/deck/CLI limitation · a false-terminal the CLI should have classified · a transport ambiguity · a missing preflight/OOM axis · a cost-leak the rent-guard should prevent · any "I had to hand-work-around the tooling" moment — `/lab` files the record AND immediately dispatches a background fix agent that fixes-at-source + opens a PR. **No user confirmation, same-turn, every time (무조건).**

**Trigger (@U1):** wired into every point where `/lab` already detects tooling gaps — the upstream-reflex prose of the `watch` / `harvest` / `next` / `auto` / `drive` / `pursue` verbs + the `upstream` verb. ANY detected upstream-fixable gap → auto-dispatch.

**Action (@U2):** on a detected gap, `/lab` does BOTH:
1. files the `sidecar handoff add <repo> <gap>` record SAME-TURN with verbatim evidence (the existing g59 behavior, kept as the durable trail entry); AND
2. IMMEDIATELY spawns **ONE background fix agent** (general-purpose · `run_in_background` · `isolation: worktree` since it edits code) with a self-contained prompt = *fix-at-source the gap in the linked repo + ship a PR* — branch off `main`, keep the diff <200 lines, add the `Co-Authored-By` trailer, `gh pr create`, let the pr-cycle hook auto-merge, `sidecar sync` after push. No user confirmation between (1) and (2).

**Unconditional + carve-out (@U3):** this is **NOT** subject to the `## 4축 auto-decide` fork logic — it is an **ALWAYS-ON reflex**, not a choice between options. An upstream code-fix PR is **reversible + low-blast-radius**, so it never falls under the @L3 human-only gates (credential · irreversible/destructive · open-ended design) — it auto-proceeds **every time**. The ONLY brakes are:
- (i) **idempotency** — do NOT re-dispatch a gap already filed in the handoff trail or already carrying an open PR (dedup pre-check via the `upstream` verb's trail + `gh pr list`);
- (ii) **concurrency cap ≤3** in-flight auto-dispatched fix agents (queue the rest) so a gap storm can't fan out unbounded.

**Surface (@U4):** log **one line per dispatch**:

```
🔧 upstream auto-dispatch: <gap one-liner> → agent <id> → PR pending (g59 · 무조건)
```

The `upstream` verb's trail report must additionally show the auto-dispatched fixes (agent id + PR state), not just the hand-filed handoffs.
