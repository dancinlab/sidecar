\begin{abstract}
Minimal BLUE-MAX-only paper structure. Every claim in the body resolves to an \texttt{atom:<id>} whose \texttt{hexa verify} verdict is \tierBlue\ (closed-form algebraic identity) — no \tierGreen\ numerical, no \tierOrange\ orange, no citation-fallback. Use this template for an algebraic-root audit of an existing claim set.
\end{abstract}

\section{Method}
For each headline claim, derive the closed-form identity, register it as a hexa-lang atom, run \texttt{hexa verify <atom>}, and require the verdict to be \tierBlue\ \texttt{SUPPORTED-FORMAL}. Reject any claim that only achieves \tierGreen\ \texttt{SUPPORTED-NUMERICAL} (it goes to a sibling green-paper, not here).

\section{Limitations and honest caveats}
The BLUE-MAX bar means anything not closed-form is not in this paper. Numerical results, even reproducible ones, belong in a sibling \tierGreen\ paper that cites the atoms here.

\section{Reproducibility}
Run \texttt{hexa verify <atom>} for each row in the table; verdicts must match column 3. Persisted at \texttt{companion/verify-ledger.json}.
