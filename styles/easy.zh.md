# Easy(友好)回复 — canonical 参考

> 面向用户回复的"友好"风格 canonical 参考。
> 通过 `/easy` 或自然语言触发（`友好一点`、`简单点说`、`Easy 模式`、…）激活。

## 适用范围 (Tier-A)

- 交互式 CLI 聊天 (Claude Code TUI)
- 带叙述的 CLI 工具 stdout / stderr
- docs / README 冷启动入口
- 错误消息 trailer 正文 (原因 + 修复行)
- 提交消息正文的 user-summary 段落 (非标题 —— 标题保持简洁)

## 不适用

- 代码标识符 / 数学符号 / API 名称 / DOI / 提交 SHA / 文件路径
- CI 机器管道输出 (`--format json` / `jsonl`)

## 7 要素模式 (gold 参考)

每个非平凡的概念解释都应命中以下 7 个要素:

1. **图标** —— 一个视觉锚定主题的表情符号 (例: 🧶 🤖 ✂️ 🦠)
2. **名称** —— canonical 标识符 (例: `HEXA-WEAVE`)
3. **昵称** —— 用户语言中简短亲切的名字 (例: `"编织 AI"`)
4. **做什么** —— 一行平实说明
5. **类比** —— 日常物品比较 (织毛衣 / 夹爪机器人 / RNA 剪刀 / 乐高足球)
6. **ASCII 图** —— 围栏 ``` ``` 块中的示意图 (树 / 并排 / 前后 / 结构草图)
7. **对比** —— 与最接近的现有工具有何不同 (vs AlphaFold / vs 单蛋白折叠)

### Gold 示例: HEXA-* 家族

```
🧶 HEXA-WEAVE — "编织 AI"

- 做什么: 把蛋白质 + DNA + 药物一次性编织，预测如何缠绕
- 类比: 用几种彩色毛线织毛衣
```

ASCII:

```
线 1  ━━━━━━━━━━━
        ╲╱╲╱╲╱       ← 多股线
线 2  ━━━━━━━━━━━      相互编织
        ╱╲╱╲╱╲        成结实的布
线 3  ━━━━━━━━━━━
```

- 对比: AlphaFold = 一次折纸，WEAVE = 编织多股线

```
🤖 HEXA-NANOBOT — "分子机械臂"

- 做什么: 设计分子如何运动 (开/合，抓/放)
```

## ASCII 结构 4 种模板 (可复制的骨架)

画 ASCII 图 (7 要素中第 6 个) 时，不要每次从零开始 —— 从下面 4 个里选形状匹配的一个，复制后填充。

### 这种形状时 → 用这个模板 (一行指南)

```
你想展示什么?                          → 哪个模板
───────────────────────────────────────────────
变化 · 改进 · "以前是 X，现在是 Y"       → 1. 前后 (before/after)
上→下 · 分叉 · 组成要素列表              → 2. 树 (tree)
把选项 A 和 B 对照比较                   → 3. 并排 (side-by-side)
各部件如何连接/流动                      → 4. 结构草图 (structure sketch)
```

### 模板 1 — 前后 (before/after)

展示变化或改进时。用箭头 `→` 分隔"前 → 后"。

```
前 (before)            后 (after)
───────────           ───────────
 ▢ 慢步骤        →      ▢ 快步骤
 ▢ 手动           →      ▢ 自动
```

### 模板 2 — 树 (tree)

展示上下层级或分解时。用 `├─` `└─` 分枝 (只有最后一枝用 `└─`)。

```
根 (整体)
├─ 枝 A
│  ├─ 叶 A1
│  └─ 叶 A2
└─ 枝 B
   └─ 叶 B1
```

### 模板 3 — 并排 (side-by-side)

把两个选项对照比较时。用 `│` 分隔中间。

```
   选项 A           │      选项 B
 ─────────────     │    ─────────────
  + 优点 1          │     + 优点 1
  − 缺点 1          │     − 缺点 1
```

### 模板 4 — 结构草图 (structure sketch)

展示各部件如何连接与流动时。用方框 `[ ]` + 箭头 `──▶` 连接。

```
[ 输入 ] ──▶ [ 处理 ] ──▶ [ 输出 ]
              │
              └──▶ [ 旁支 ]
```

## 普通人翻译检查清单

写说明前快速过一遍的 5 步:

```
[ ] 1. 检测术语 —— 有缩写(API·CPU) · 数学符号(σ·∑·∂) · 代码/产品名吗?
[ ] 2. 换成生活用语 —— 每个术语换成了日常词或类比吗? (换不掉就在首次出现处展开)
[ ] 3. 应用 7 要素 —— 每个非平凡概念都具备 7 要素(图标…对比)吗?
[ ] 4. ASCII ≥1 —— 在形状重要处用上述 4 种模板之一放了图吗?
[ ] 5. 重读 —— 初次接触该领域的人能一遍读懂吗?
```

## 反例 (何时不适用)

- 含标识符 / 数学符号的代码块
- CI 机器管道 JSON / JSONL 输出
- 无叙述的纯代码输出
- 已声明理由的紧急安全警报 (严重度正当时允许 5 连表情)
